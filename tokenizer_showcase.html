Tokenizer Anatomy

hfl/chinese-roberta-wwm-ext
到底怎么分词

这个模型并不是先做传统中文分词再整词喂给 BERT。它使用的是 BertTokenizer 的三段式处理：BasicTokenizer 先按中文字符与标点做基础切分， 再由 WordPiece 用最长匹配优先策略映射到词表，最后加上 [CLS]、[SEP] 等特殊 token。

一句话理解

中文 BERT 系列里，很多汉字本身就是有效 token。WWM 指的是预训练时“整词掩码”的策略， 不是说推理阶段一定先做中文词语分词。运行时的核心仍然是 字符级基础切分 + WordPiece 子词映射。

1 先给 CJK 字符两边加空格，让中文在基础阶段近似按字切开。
2 再用 WordPiece 的最长匹配优先算法查词表，而不是用 jieba 这类传统分词器。
3 WWM 是预训练掩码策略，用来让同一个词的多个字一起被 mask。

运行时真正发生的事

输入中文句子时，模型不会先做“词法级别的语义分词”，而是先做规则切分，再做词表映射。

BertTokenizer → BasicTokenizer → WordPiece → Special Tokens

三层流程

下面这三步，就是这个 tokenizer 的核心路径。你后面训练时看到的 token 长度、chunk 长度、截断行为，本质上都由这三步决定。

01

BasicTokenizer

处理大小写、空白符、标点，并对中文 CJK 字符做显式切开。你可以近似理解成“先把句子拆成更小、更规整的基础片段”。

tokenize_chinese_chars punctuation split whitespace cleanup
02

WordPiece

对每个基础片段做词表匹配，采用最长匹配优先。很多常见中文单字会直接命中词表；罕见片段可能被拆成更细的子词，最坏退化成 [UNK]。

greedy longest-match-first vocab lookup subword fallback
03

Model Input

在 token 序列首尾加入 [CLS] 和 [SEP]，再把它们转成 id。后面的 attention、截断、padding、chunking 都是在这个序列层面发生。

这不是传统中文分词

如果你脑子里想到的是“我 / 爱 / 自然语言处理”这种词级分词，那要及时纠正。这个 tokenizer 运行时并不依赖 jieba、THULAC 这类词法工具。

  • 运行阶段更接近“字符规整 + 子词映射”
  • 很多常见汉字会直接作为 token
  • 模型最终理解的是 token id 序列，不是人类肉眼看到的中文词边界

WWM 真正指什么

Whole Word Masking 主要发生在预训练时。它的意思是：如果一个中文词由多个字组成，那么被 mask 时倾向一起 mask，而不是随机只盖住其中一个字。

  • 它提升的是预训练策略
  • 不是说推理时一定先切成完整中文词
  • 你现在做分类训练时，实际仍是走 BertTokenizer 的正常 tokenization 流程
